## 1. Context & Goal

### What We're Testing
The CMA Automation System's ability to process a real Indian manufacturing company's financials (Dynamic Air Engineering) and produce a correct CMA Excel file — covering the full pipeline: document upload → extraction (including scanned PDF via Claude Vision OCR) → classification (manufacturing industry rules) → Excel generation.

### Why Dynamic Air Engineering
- **Different industry from Mehta Computers:** Mehta was Trading. Dynamic is Manufacturing/Engineering. Different classification rules, different CMA rows that matter.
- **Mixed document types:** Excel (FY22), text PDFs (FY23), scanned 33-page PDF (FY24), provisional Excel (FY25). This tests every extraction path.
- **Phase 10 validation:** FY24 is a fully scanned audited PDF — the first real-world test of the Claude Vision OCR pipeline built in Phase 10.
- **Ground truth exists:** `CMA Dynamic 23082025.xls` was manually prepared by the CA firm. We can compare output precisely.

### Success Definition
1. App extracts >50 line items from FY24 scanned PDF via Vision OCR
2. Classification accuracy ≥ 85% against ground truth CMA values
3. Final generated CMA matches known values for 10+ key rows within 1%
4. Three-way comparison (Claude scan vs App extraction vs Existing CMA) shows extraction accuracy ≥ 90% for text-based years (FY22, FY23, FY25)

## 4. Agent Specifications

### Pre-Test Agent P1: Document Scanner
**Runs:** Before main test, in background
**Inputs:** FY22 Excel + FY22 PDFs + FY23 PDFs + FY25 Excel
**Task:** Read every document directly (not through app) and extract all financial line items
**Output:** `test-results/dynamic/dynamic_ground_truth.md`
**Note:** FY24 (scanned PDF) is SKIPPED — cannot be read without OCR. FY24 ground truth comes from CMA Dynamic file only.

### Pre-Test Agent P2: CMA Analyzer
**Runs:** Before main test, in background
**Inputs:** `CMA Dynamic 23082025.xls`
**Task:** Extract ALL values from INPUT SHEET, organized by row number and year
**Output:** `test-results/dynamic/dynamic_cma_known_values.json` + `dynamic_cma_analysis.md`
**Key rows to capture:** R22 (Sales), R42 (Raw Materials), R45 (Factory Wages), R48 (Power), R56 (Depreciation), R67 (Salary), R68 (Rent), R83 (Interest TL), R85 (Bank Charges), plus all balance sheet rows

### Agent 1: Infrastructure Health
**Runtime:** ~5 minutes
**Checks:**
- Docker containers: backend + frontend + redis + worker — all running?
- Backend health: `GET http://localhost:8000/health` → 200
- Worker startup: `docker compose logs worker | tail -20` → "Starting worker for 3 functions" (no import errors for pdf2image, anthropic)
- Redis: `docker compose exec redis redis-cli ping` → PONG
- Phase 10 dependencies: `docker compose exec backend python -c "from pdf2image import convert_from_bytes; print('OK')"` → OK
- Anthropic client: `docker compose exec backend python -c "import anthropic; print('OK')"` → OK

**Gate:** ALL checks must pass before proceeding to Agent 2.
**Output:** `test-results/dynamic/agent1-health.json`

### Agent 2: Document Intelligence
**Runtime:** ~10 minutes
**Task:** Analyze each document file (before uploading) and produce a manifest
**Checks:**
- Confirm file exists and is readable for each document in the manifest
- For FY24 PDF: run `docker compose exec backend python -c "import pdfplumber; pdf = pdfplumber.open(...); print(len(pdf.pages), 'pages'); print(pdf.pages[0].extract_text()[:200])"` — should return 33 pages, text should be empty/garbled (confirming it's scanned)
- Identify industry from FY25 provisional Excel (look for manufacturing-type line items: wages, power, raw materials)
- Confirm file sizes are reasonable (FY24 PDF should be several MB for scanned)

**Output:** `test-results/dynamic/agent2-manifest.json`

```json
{
  "company": "Dynamic Air Engineering",
  "industry": "Manufacturing",
  "years_covered": ["FY22", "FY23", "FY24", "FY25"],
  "documents": [
    {"file": "BS-Dynamic 2022 - Companies Act.xlsx", "year": "FY22", "type": "Excel", "extractor": "ExcelExtractor", "readable": true},
    {"file": "Audited Financials FY-2024 (1).pdf", "year": "FY24", "type": "ScannedPDF", "pages": 33, "extractor": "OcrExtractor", "text_extractable": false}
  ]
}
```

### Agent 3: Extraction Pipeline
**Runtime:** 30-90 minutes (FY24 Vision OCR is the slow step — 33 pages @ ~15 pages/batch = 3 API calls)
**This is the most important agent.**

**Setup:**
1. Create client via API: `POST /api/clients/` — name: "Dynamic Air Engineering", industry: "manufacturing"
2. Create CMA report: `POST /api/cma-reports/` — linked to client, years: FY22-FY25
3. For each document (in order):
   a. Upload document: `POST /api/documents/upload`
   b. Trigger extraction: `POST /api/documents/{id}/extract`
   c. Poll until complete: `GET /api/documents/{id}/status`
   d. Fetch extracted items: `GET /api/documents/{id}/line-items`
   e. Record: count of items, any errors, time taken

**Special Validation for FY24 (Vision OCR):**
- Check `docker compose logs worker` after extraction — should contain "OcrExtractor", "Vision OCR", "Batch 1-15", "Batch 16-30"
- Item count must be > 50 (a 33-page audited document should yield 50-200 items)
- Check for ambiguity_question fields — manufacturing docs often have combined line items
- If item count = 0: STOP and report — Vision OCR failed

**Metrics to Record:**
```
FY22 Excel:      target > 20 items
FY22 Notes PDF:  target > 15 items
FY23 PDFs:       target > 30 items
FY24 Scanned:    target > 50 items ← KEY TEST
FY25 Excel:      target > 20 items
Total:           target > 135 items
```

**Verification Step (MANDATORY per CLAUDE.md):**
- Call verify endpoint for all items before classification
- Record: total verified, any ambiguity questions shown

**Output:** `test-results/dynamic/agent3-extraction.md`

### Agent 4: Classification Quality
**Runtime:** ~20 minutes
**Inputs:** Extracted line items from Agent 3 + `dynamic_cma_known_values.json` from P2

**Task:** Trigger classification for all extracted items, then validate results.

**Manufacturing-Specific Validation:**
Check that these CMA rows get populated (they should be non-zero for a manufacturing company):

| Row | Field | Expected Behaviour |
|-----|-------|--------------------|
| R22 | Domestic Sales | Primary revenue line — must match |
| R42 | Raw Materials (Indigenous) | Raw materials purchased for manufacturing |
| R45 | Factory Wages / Direct Wages | Workers on factory floor |
| R48 | Power, Coal, Fuel | Electricity for machines |
| R49 | Other Manufacturing Expenses | Misc factory costs |
| R56 | Depreciation (Manufacturing) | Depreciation on plant & machinery |
| R67 | Salary & Staff | Admin/office salaries |
| R83 | Interest on Term Loans | Bank loan interest |

**Metrics:**
- Tier 1 (fuzzy/learned): target > 30%
- Tier 2 (AI): target 20-40%
- Tier 3 (doubt): target < 30% (manufacturing is more complex than trading — slightly higher doubt acceptable)
- Accuracy against known CMA values: target ≥ 85%

**Comparison Method:** For each key CMA row, compare the classified amount vs the known value from `dynamic_cma_known_values.json`. Calculate % match.

**Output:** `test-results/dynamic/agent4-classification.md`

### Agent 5: Excel Generation & Validation
**Runtime:** ~10 minutes
**Task:** Generate CMA Excel, download it, compare against existing CMA.

**Steps:**
1. Trigger Excel generation: `POST /api/cma-reports/{id}/generate`
2. Poll until complete
3. Download: `GET /api/cma-reports/{id}/download`
4. Save to: `test-results/dynamic/generated_dynamic_cma.xlsm`
5. Read INPUT SHEET from generated file using Python
6. Read INPUT SHEET from `CMA Dynamic 23082025.xls`
7. Compare row-by-row for all 4 years

**Validation Table:**
```
For each row in INPUT SHEET:
  Match if |generated_value - known_value| / known_value < 1%  ← ✅
  Flag if difference 1-10%                                      ← ⚠️
  Error if difference > 10%                                     ← ❌
```

**Output:** `test-results/dynamic/agent5-excel.md` + `generated_dynamic_cma.xlsm`

### Agent 6: E2E Critical Path (Abbreviated)
**Runtime:** ~15 minutes
**Focus:** The specific steps that failed in Mehta — fix confirmed?

**Journey (9 steps, not full 15):**
1. Navigate to app → login
2. Find Dynamic Air Engineering client
3. Navigate to CMA report
4. Upload one document (FY25 Excel — fastest)
5. Trigger extraction → wait for complete
6. Open verification screen — see line items?
7. Approve items → trigger classification
8. Click "Generate CMA" button (this failed in Mehta — confirm fixed)
9. Download Excel file (this failed in Mehta — confirm fixed)

**Pass criteria:** Steps 8 and 9 must both succeed (these were the Mehta failures).

**Output:** `test-results/dynamic/agent6-e2e.md`

### Agent 7: Three-Way Comparison & Final Report
**Runtime:** ~15 minutes
**Inputs:** ALL previous agent outputs
**This agent runs last and assembles the complete picture.**

**Three-Way Comparison Logic:**

```
COMPARISON 1: Extraction Accuracy
  For FY22, FY23, FY25 (text-readable documents):
    Claude scan (ground_truth.md) → "true" line items
    App extraction (agent3-extraction.md) → extracted items
    Match: description fuzzy match + amount within 1%
    Score: matched / total_in_ground_truth × 100 = Extraction Accuracy %

COMPARISON 2: Classification Accuracy
  For all years:
    App classified CMA rows → agent4-classification.md
    Known CMA values → dynamic_cma_known_values.json
    Score: matched_rows / total_key_rows × 100 = Classification Accuracy %

COMPARISON 3: End-to-End Accuracy
  Generated CMA (agent5-excel.md) vs CMA Dynamic 23082025.xls
  Row-by-row comparison for all 4 years
  Score: rows_within_1% / total_rows × 100 = End-to-End Accuracy %
```

**Output:**
- `test-results/dynamic/agent7-comparison.md` — detailed delta report
- `test-results/dynamic/DYNAMIC_TEST_REPORT.md` — final assembled report

## 7. Pass/Fail Gates

| Gate | Condition | Action if Fail |
|------|-----------|---------------|
| G1: Infrastructure | All Docker containers running, pdf2image + anthropic importable | STOP — fix before proceeding |
| G2: FY24 Scanned | Vision OCR produces > 50 items | STOP — Phase 10 bug, investigate |
| G3: Total Extraction | > 135 items total across all documents | WARNING — proceed but flag |
| G4: Classification | Accuracy ≥ 85% on key rows | WARNING — improvement rules needed |
| G5: Excel Match | Generated CMA ≥ 80% rows within 1% | ERROR — classification or Excel generation bug |
| G6: E2E | Steps 8 + 9 (generate + download) succeed | ERROR — critical regression |

## 8. Risks & Mitigations

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| FY24 Vision OCR slow (33 pages × cost) | High | Budget ~15 min; log batch progress |
| Manufacturing classification doubt rate high | Medium | Manufacturing has more complex items; target <30%, not <15% |
| Excel comparison fails if scale differs (lakhs vs absolute) | Medium | Normalize all values to lakhs before comparison |
| FY22 Excel has unusual sheet structure | Low | Agent 2 inspects sheets before upload |
| CMA Dynamic file has different row mapping than app | Low | Agent P2 maps rows explicitly |
| pdf2image crashes on 33-page PDF | Low | Phase 10 tested this; batching at 15 pages handles it |
