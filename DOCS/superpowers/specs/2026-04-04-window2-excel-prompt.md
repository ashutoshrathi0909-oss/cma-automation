### Comparison logic

```python
def compare_cell(expected, actual, tolerance=0.02):
    """Returns (status, diff_pct)"""
    if expected is None or expected == "" or expected == 0:
        if actual is None or actual == "" or actual == 0:
            return ("BOTH_EMPTY", 0)
        return ("EXTRA", None)  # Generated has value, GT doesn't
    
    if actual is None or actual == "" or actual == 0:
        return ("MISSING", None)  # GT has value, generated doesn't
    
    if not isinstance(expected, (int, float)) or not isinstance(actual, (int, float)):
        return ("NON_NUMERIC", None)
    
    if abs(expected) < 0.01 and abs(actual) < 0.01:
        return ("MATCH", 0)  # Both effectively zero
    
    if abs(expected) < 0.01:
        return ("MISMATCH", float('inf'))
    
    diff_pct = abs(actual - expected) / abs(expected)
    if diff_pct <= tolerance:
        return ("MATCH", diff_pct)
    return ("MISMATCH", diff_pct)
```

## PHASE 6: MISMATCH ROOT CAUSE ANALYSIS

For every **MISMATCH** or **MISSING** cell:

1. **Check unit conversion first**:
   - FY2021 (col B): source is rupees, expected divisor = 10,000,000
   - FY2022 (col C): source is lakhs, expected divisor = 100
   - FY2023 (col D): source is rupees, expected divisor = 10,000,000
   - If the mismatch is exactly 100x or 100,000x off → unit conversion bug

2. **Check if item was classified elsewhere**:
   - Query classifications: is there a line item that SHOULD be in this row but was classified to a different row?
   - Example: "Carriage Outward" might be in Row 71 (Others Admin) instead of Row 70 (Selling)

3. **Check if item was extracted**:
   - Was the source line item even extracted from the document?
   - If missing from extraction → extraction error

4. **Check if doubt-resolved**:
   - Read `doubt-resolutions.json`
   - If this row involves a doubt that Opus resolved → note the decision and whether it was correct

5. **Classify the error**:

| Code | Type | How to Detect |
|------|------|---------------|
| A | Synonym Miss | Item exists but classified to wrong row with similar name |
| B | Industry Context | Classification depends on industry (manufacturing vs services) |
| C | Conditional | Amount or context determines the correct field |
| D | Aggregation | Multiple items should net to one cell but were split |
| E | Correct Doubt | AI rightly flagged uncertainty, Opus decided |
| F | Unit Error | Value is exactly Nx off (N = conversion factor) |
| G | Extraction Error | Source amount wrong or item not extracted |
| H | Misroute | Item classified to completely wrong section |
| I | Missing Item | Line item not in extraction at all |
